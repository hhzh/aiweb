# 文章优化器

基于6种技术文章写作框架和通用写作技巧，对现有文章进行诊断、优化、输出。

## 一、输入规范

用户输入格式：`<文章路径> [优化要求]`

- **文章路径**（必填）：Markdown 文件的绝对或相对路径
- **优化要求**（可选）：如"更通俗"、"加案例"、"精简到3000字"、"面向新手"等

示例：
- `@src/claudecode/01-guide.md 优化一下，更通俗一些`
- `/path/to/article.md 帮我补充实战案例，精简到3000字`
- `article.md 改写成入门科普风格`

## 二、6种写作框架速查

优化前先判断文章属于哪种类型，对照对应框架检查完整性：

| 类型 | 核心目标 | 框架步骤 |
|-----|---------|---------|
| 入门科普 | 降低门槛，建立认知 | 场景引入→核心定义→核心价值→基础应用→总结延伸 |
| 实战教程 | 手把手教学，解决问题 | 需求明确→前置准备→分步操作→结果验证→常见问题→拓展优化 |
| 深度解析 | 深入底层，理解本质 | 引言(背景+问题)→原理拆解→源码/案例佐证→优缺点分析→总结展望 |
| 问题排查 | 定位问题，给出方案 | 问题描述→问题定位→原因分析→解决方案→预防措施 |
| 对比选型 | 对比分析，辅助决策 | 选型背景→对比对象与指标→详细对比→选型建议→总结 |
| 经验总结 | 复盘总结，传递技巧 | 实战背景→核心问题与方案→经验提炼→踩坑教训→总结反思 |

## 三、通用写作技巧检查清单

无论哪种类型，都要逐项检查：

### 开篇（3秒抓住注意力）
- [ ] 是否在3句话内切中读者痛点或兴趣点？
- [ ] 是否避免了冗长铺垫（"在当今社会..."式开头）？
- [ ] 优先用4种方法之一：痛点切入、疑问开头、场景代入、价值前置

### 中间内容
- [ ] 分点是否清晰、格式是否统一（数字+关键词开头）？
- [ ] 每个观点是否配有具体案例（而非空洞说教）？
- [ ] 段落之间是否有过渡句衔接？
- [ ] 是否有逻辑跳跃或断层？

### 结尾
- [ ] 是否总结了核心价值？
- [ ] 是否给出了行动引导或延伸方向？
- [ ] 是否避免了草草收尾或新增无关内容？

### 细节
- [ ] 是否删除了多余修饰词（"非常、很、极其"）？
- [ ] 是否用具体数据替代了模糊词（"超过80%"而非"很多人"）？
- [ ] 是否避免了口语化和网络流行语？
- [ ] 长句是否拆为短句？

### 文章结构
- [ ] 整体是否采用了合适的结构（总分总/问题-分析-解决/并列/递进）？
- [ ] 首尾是否呼应？

## 四、执行流程

### 步骤1：读取并分析文章

1. 用 Read 工具读取文章全文
2. 识别文章类型（入门科普/实战教程/深度解析/问题排查/对比选型/经验总结）
3. 对照对应框架检查结构完整性，标记缺失或薄弱的环节
4. 用写作技巧检查清单逐项扫描，标记问题点
5. 用 Markdown 排版规范（第五章）逐项扫描，标记排版问题
6. 根据用户额外的优化要求，标记需要调整的内容

输出诊断报告：

```
### 文章诊断报告

**文章类型**：{类型}
**匹配框架**：{框架名称}
**当前字数**：{字数}

**结构完整性**：
- ✅ {已具备的步骤}
- ❌ {缺失的步骤}
- ⚠️ {存在但薄弱的步骤}

**写作技巧问题**：
- {问题1}
- {问题2}
- ...

**用户要求调整**：
- {要求1}

**排版问题**：
- {排版问题1}
- {排版问题2}
```

### 步骤2：制定优化策略

根据诊断结果，制定优化方案。优化不是重写，而是在保留原文核心内容的基础上改进：

| 问题类型 | 优化策略 |
|---------|---------|
| 缺失框架步骤 | 补充对应步骤的内容，风格与原文一致 |
| 开篇不够吸引 | 用痛点切入/疑问/场景代入/价值前置重写开头 |
| 案例不足 | 为缺少案例的观点补充具体案例（1-2句话的简短案例即可） |
| 结构混乱 | 重新组织段落顺序，添加过渡句 |
| 内容冗长 | 删除重复表述，合并相似段落，精简修饰词 |
| 口语化严重 | 替换为专业但不生硬的表述 |
| 模糊表述 | 用具体数据或场景替代 |
| 结尾薄弱 | 补充总结+行动引导 |
| 代码块缩进 | 移除缩进，列表内代码块改用加粗标题+独立代码块格式 |
| 代码块语言标记错误 | 按第五章规则统一修正 |
| 注释风格不统一 | 统一改为 `#` 注释符 |
| 代码示例英文描述 | 改为中文描述，保留代码本身的英文 |
| 引用不存在的命令 | 替换为通用方案并确认真实存在 |

### 步骤3：执行优化

用 Edit 工具逐步修改文章。优先处理结构性问题，再处理细节问题。

**优化原则**：
- 保留原文的核心观点和信息，不改变文章主旨
- 保持原文的语言风格（除非用户要求改变风格）
- 每次修改聚焦一个问题，避免大段重写
- 如果用户指定了字数目标，通过精简或补充调整篇幅

**注意 frontmatter**：如果原文有 YAML frontmatter（`---` 之间的内容），不要修改或删除它。

### 步骤4：输出优化报告

优化完成后，输出报告：

```
### 优化完成

**优化项**：
1. {优化项1：做了什么，为什么}
2. {优化项2：做了什么，为什么}
3. ...

**框架补全**：
- ✅ 已补充：{补充的步骤}

**写作技巧改进**：
- ✅ {改进项}

**排版改进**：
- ✅ {排版改进项}

**字数变化**：{原字数} → {新字数}
```

## 五、Markdown 排版规范（VuePress 兼容）

优化文章时，必须同时检查排版问题，确保 VuePress 正确渲染。以下是强制规则：

### 代码块规则

1. **代码块标记紧贴行首**：` ``` ` 必须从行首开始，不得缩进。缩进的代码块在 VuePress 中会被视为代码内容而非 Markdown 语法，导致渲染错乱。
   - 列表内的代码块，改用 `**1. 标题**` 加粗标题 + 独立代码块的格式，避免缩进
   - 正确示例：
     ```text
     **1. 精准引用文件**

     ```text
     > 解释 @src/utils/auth.js 中的逻辑
     ```
     ```
   - 错误示例（代码块缩进在列表内）：
     ```text
     1. 精准引用文件：

        ```text
        > 解释逻辑
        ```
     ```

2. **代码块语言标记统一**：
   - 交互式对话示例（`>` 提示符）：使用 ` ```text `
   - 纯 shell 命令（可实际执行）：使用 ` ```bash `
   - JSON 配置：使用 ` ```json `
   - Markdown 示例本身：使用 ` ```markdown `
   - Mermaid 图表：使用 ` ```mermaid `
   - 禁止对交互式对话示例使用 ` ```bash ` 或 ` ```markdown `

3. **注释风格统一**：
   - ` ```text ` 代码块中：使用 `#` 作为注释符
   - ` ```bash ` 代码块中：使用 `#` 作为注释符
   - 禁止在 ` ```text ` 或 ` ```bash ` 中使用 `//` 作为注释符

### 列表与编号规则

4. **有序列表避免编号歧义**：当列表项包含代码块时，使用 `**1. 标题**` 加粗格式代替 `1. 标题`，避免 Markdown 解析器编号错误（VuePress/markdownlint 会因代码块打断列表而报编号警告）

5. **避坑/问题类编号**：使用 `**问题1：标题**` 格式，而非 `1. **问题1：标题**`，避免编号与内容层级混淆

### 语言与内容规则

6. **代码示例中文化**：交互式示例中的指令和注释尽量使用中文描述（如 `> 解释鉴权逻辑` 而非 `> Explain the auth logic`），适合中文读者阅读。实际代码（函数名、变量名、命令参数）保持英文不变

7. **不存在的命令禁止出现**：引用工具/框架的命令时，必须确认该命令真实存在。如果不确定，用通用方案替代（如 Docker 隔离替代不存在的 `/sandbox` 命令）

## 六、特殊场景处理

### 用户要求切换文章类型

如"把这篇入门教程改写成深度解析"：
1. 按目标框架重新组织结构
2. 调整内容深度和语言风格
3. 补充目标框架需要的步骤内容
4. 精简原文中不属于目标框架的部分

### 用户要求精简字数

1. 先删除冗余表述和重复内容
2. 合并相似段落
3. 精简案例（保留核心论据，删除过程细节）
4. 最后检查结构完整性是否受影响

### 用户要求补充特定内容

如"补充实战案例"、"加一个对比表格"：
1. 在对应位置插入内容
2. 保持与原文风格一致
3. 添加过渡句保证衔接自然

## 七、常见优化模板

### 重写开篇（痛点切入型）

```
你是否遇到过{痛点场景}？{简述痛点带来的影响}。其实，{核心概念/工具}可以帮你轻松解决这个问题。本文将{文章价值承诺}，{适用人群暗示}也能快速上手！
```

### 重写开篇（价值前置型）

```
读完这篇文章，你能掌握{核心收获1}、{核心收获2}，{行动结果}。{简要背景}，本文{写作方式}，{读者保障}。
```

### 补充过渡句

```
掌握了{上节内容}，接下来我们看看{下节主题}——{下节核心价值}。
```

### 强化结尾

```
{1-2句总结核心要点}。建议{行动引导}，{预期效果}。后续可以{延伸方向}，{进一步价值}。
```
