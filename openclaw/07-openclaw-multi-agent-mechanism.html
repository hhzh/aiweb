OpenClaw 多智能体核心机制详解

一个 Gateway 进程运行多个完全隔离的智能体——按消息来源精准路由、按任务类型并行分治、按组织需求委托代理。OpenClaw 多智能体体系从个人双账号分流到团队专家通道分工，再到组织级委托数字员工，均可通过标准化配置实现。本文聚焦隔离架构、路由规则、并行通道、状态感知、委托代理五大核心概念，详细配置与实战示例请参考 多智能体路由详解。

一、多智能体核心定位与隔离架构

1.1 什么是一个独立智能体

OpenClaw 中一个智能体是完整隔离的决策单元，拥有独占的运行环境：

  • 独立工作区：专属文件、人格配置（AGENTS.md/SOUL.md）、本地技能
  • 独立状态目录：认证配置、模型注册表、智能体配置
  • 独立会话存储：聊天历史、路由状态、上下文数据
  • 独立认证：不共享主智能体凭证，如需复用需手动复制配置文件

1.2 核心隔离铁律

  • 严禁跨智能体复用 agentDir，会导致认证 / 会话冲突
  • 凭证共享需手动复制 auth-profiles.json 到目标智能体目录
  • 私聊会合并为智能体主会话，真隔离需一人一智能体

完整文件路径体系、单智能体默认模式与生命周期管理，请参考 多智能体路由详解 第一章与第二章。

二、智能体路由规则

多智能体的核心是消息绑定路由，遵循最精确匹配优先原则，所有匹配条件为AND 逻辑。匹配优先级从高到低：

  1. 精确用户 / 群组 / 频道（peer）匹配
  2. 线程父会话（parentPeer）匹配
  3. Discord 服务器 + 角色（guildId+roles）匹配
  4. Discord 服务器（guildId）匹配
  5. Slack 团队（teamId）匹配
  6. 渠道账号（accountId）匹配
  7. 全渠道（channel:"*"）匹配
  8. 回退默认智能体

绑定配置语法、五大典型路由场景与全局私聊准入控制，请参考 多智能体路由详解 第三章。

三、并行专家通道

并行专家通道用于解决资源竞争问题，将不同类型任务分配给专属智能体通道，避免阻塞。核心设计原则：

  • 会话锁：同一会话仅允许单次运行
  • 模型容量：全局共享提供商配额
  • 工具容量：Shell / 浏览器等为物理瓶颈
  • 上下文预算：长会话降低后续响应速度

落地分三阶段：通道契约 → 优先级与并发控制 → 协调器模式。

完整三阶段落地法、配置示例与核心价值，请参考 多智能体路由详解 第四章。

四、在线状态（Presence）感知

Presence 是网关与客户端的轻量在线感知系统，用于展示连接状态与设备信息。核心特性：

  • 合并去重：以 instanceId 为唯一键
  • 自动过期：超过 5 分钟的条目自动清理
  • 数量限制：最大存储 200 条

完整状态来源、字段定义与调试方法，请参考 多智能体路由详解 第五章。

五、企业级委托代理（Delegate）

委托架构将多智能体从个人场景扩展到组织级部署，智能体以独立身份代用户执行操作，不模拟真人身份。三级能力模型：

  1. Tier1 只读 + 草稿：仅读取数据、生成草稿，需人工确认
  2. Tier2 代发：可代发消息、创建日历，显示代理标识
  3. Tier3 主动执行：定时任务自主运行，异步交付结果

完整安全加固、部署步骤与组织助理配置示例，请参考 多智能体路由详解 第六章。

六、单智能体维度：沙箱与工具独立管控

OpenClaw 支持按智能体配置沙箱与工具策略，实现精细化权限隔离。

6.1 沙箱独立配置

{
  "id": "family",
  "sandbox": {
    "mode": "all",
    "scope": "agent",
    "docker": { "setupCommand": "apt update && apt install -y git" }
  }
}

6.2 工具独立管控

支持 allow/deny 列表，独立于全局工具策略：

{
  "id": "family",
  "tools": {
    "allow": ["read", "session_status"],
    "deny": ["exec", "write", "browser"]
  }
}

智能体间通信（Agent-to-Agent）配置，请参考 多智能体路由详解 第七章。

七、常用 CLI 命令

# 创建新智能体
openclaw agents add work
# 列出智能体与绑定
openclaw agents list --bindings
# 查看智能体状态
openclaw agent status

完整 CLI 命令速查表，请参考 多智能体路由详解 第九章。

八、多智能体落地最佳实践

  1. 绝对隔离：不共享 agentDir，避免认证与会话冲突
  2. 最小权限：委托智能体从最低权限开始，按需升级
  3. 并行通道：先定义通道契约，再配置并发，最后启用协调器
  4. 路由精确：优先使用 peer 精确匹配，减少路由歧义
  5. 安全优先：群组 / 委托智能体强制开启沙箱与工具白名单

完整安全规范与审计要求，请参考 多智能体路由详解 第十章。
