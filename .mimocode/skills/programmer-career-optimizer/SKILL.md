# 程序员职场与面试文章深度优化指南

接收一个 Markdown 职场/面试文章文件路径，按以下流程完成优化并直接覆写原文件。

## 第一步：读取与画像
读取 `$ARGUMENTS` 指定的文件。
1.  **分类识别**：判断文章属于以下哪种类型：
    -   🏢 **生存篇**（年终总结、工作沟通、升职加薪）
    -   🎓 **面试篇**（面试流程、反问环节、自我介绍）
    -   🧠 **思维篇**（学生思维、职业成长、技术管理）
2.  **受众定位**：识别目标读者是“校招学生”、“初级开发”还是“中年危机程序员”。

## 第二步：多维诊断与增强
在检查基础结构的同时，重点注入以下“硬核”内容：

### 1. 结构与逻辑 (Structure)
-   **STAR 原则强化**：如果是面试题或案例分析，检查是否包含 STAR（情境、任务、行动、结果）。如果缺失，强制补充。
-   **章节修正**：修正编号，确保逻辑流为：`痛点场景` -> `错误示范` -> `核心心法` -> `实操案例` -> `总结`。

### 2. 内容丰富度 (Richness) - 【核心优化点】
-   **注入真实案例 (Case Study)**：
    -   不能只有理论。例如讲“年终总结”，必须补充一个虚构但真实的**“反面教材 vs 正面模板”**对比。
    -   *规则*：针对核心观点，补充一个具体的程序员工作场景（如：被产品经理甩锅、需求频繁变更）。
-   **补充“程序员专用”话术**：
    -   针对沟通类文章，补充具体的**“话术模板”**。
    -   *示例*：将“要学会拒绝”优化为“三步拒绝法话术：肯定需求 -> 亮出排期 -> 提供替代方案”。
-   **避坑指南**：
    -   提取文中的隐性风险，整理成**“面试/职场必死 5 大雷区”**清单。

## 第三步：视觉与交互优化
为了让读者更容易消化枯燥的职场理论，执行以下操作：

### 1. 植入 Mermaid 思维导图
在文章开头或核心章节，插入思维导图，帮助读者建立知识体系：
-   **布局**：使用 `mindmap` 或 `graph TD`。
-   **内容**：展示“程序员晋升能力模型”或“面试全流程拆解”。

### 2. 格式化与高亮
-   **对比表格**：对于“学生思维 vs 成人思维”、“初级 vs 高级 沟通方式”，必须使用**左右对比表格**。
-   **核心提示**：使用引用块标记关键心法：
    > **💡 核心心法**：面试不是考试，面试是相亲。不要只背答案，要展示你的思考过程。

## 第四步：落地总结生成
在文章末尾强制添加或优化以下内容：

### 1. 实操清单 (Action List)
1.  **话术抄作业**：提供一段可以直接复制粘贴的自我介绍/拒绝需求/争取加薪的话术。
2.  **自查表**：提供一个“面试/汇报前 10 分钟检查清单”。
3.  **资源包**：如果涉及简历或总结，提供一个“模版下载”的文本示意。

### 2. 核心对比表
| 场景 | 错误做法 (学生思维) | 正确做法 (职场高手) | 核心差异 |
| :--- | :--- | :--- | :--- |
| **需求评审** | “这个技术实现不了” | “这个需求可以用 A 方案快速上线，B 方案更完美但需 2 周” | **提供选项而非判断题** |
| **遇到 Bug** | 埋头苦干 3 天 | 1 小时定位，解决不了立刻拉群求助 | **时间成本意识** |

## 第五步：文件覆写与报告
1.  **清理**：删除所有本地绝对路径图片。
2.  **覆写**：将优化后的 Markdown 内容写回原文件。
3.  **输出报告**：向用户输出一份简洁的诊断报告。
