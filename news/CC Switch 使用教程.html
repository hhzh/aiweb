CC Switch 使用教程

一、工具概述

CC Switch 是一款跨平台桌面工具，基于 Tauri 2 + Rust 开发，专为 Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw、Hermes Agent 六大 AI 编程命令行工具打造，实现一键切换 API 供应商、统一管理配置、MCP、Skills 与提示词，彻底告别手动编辑 JSON/TOML/ENV 配置文件的繁琐操作。

官方渠道：唯一官网为 https://ccswitch.io，开源仓库地址：https://github.com/farion1231/cc-switch。

核心优势

  • 一站式管理：单界面管控 6 大 AI CLI，无需切换工具；

  • 50+ 预设供应商：覆盖官方、AWS Bedrock、NVIDIA NIM、国内大模型（通义千问、Kimi 等）；

  • 零配置切换：一键启用供应商，Claude Code 支持热切换（无需重启终端）；

  • 统一扩展管理：集中配置 MCP 服务器、Skills 插件、全局提示词；

  • 跨平台支持：适配 Windows/macOS/Linux，提供桌面端与 Web 端两种部署模式；

  • 安全可靠：配置本地 SQLite 加密存储，自动备份防丢失。

二、安装指南

2.1 系统要求

  • Windows：Windows 10 及以上；

  • macOS：macOS 12（Monterey）及以上；

  • Linux：Ubuntu 22.04+/Debian 11+/Fedora 34+ 等主流发行版。

2.2 下载安装

Windows

  1. 访问 CC 开源仓库（https://github.com/farion1231/cc-switch）或官网（[ccswitch.io](ccswitch.io)）；

  2. 进入仓库 Releases 页面，下载 CC\-Switch\-v\{version\}\-Windows\.msi（安装版）或便携版 ZIP；

  3. 安装版双击安装，便携版解压后直接运行 CC Switch\.exe。

macOS（推荐 Homebrew）

# 安装
brew install --cask cc-switch
# 更新
brew upgrade --cask cc-switch

手动安装：从仓库 Releases 页面下载 \.dmg 镜像，拖入应用程序；首次运行提示 “无法验证开发者”，前往系统设置→隐私与安全性→仍要打开。

Linux

  • Debian/Ubuntu：仓库 Releases 页下载 \.deb 包，sudo dpkg \-i 文件名\.deb；

  • Fedora/RHEL：仓库 Releases 页下载 \.rpm 包，sudo dnf install 文件名\.rpm；

  • Arch Linux：通过 AUR 安装 paru \-S cc\-switch\-bin；

  • 通用：仓库 Releases 页下载 AppImage，赋予执行权限后直接运行。

2.3 首次启动初始化

  1. 启动后自动创建数据目录 \~/\.cc\-switch/，包含数据库、备份、Skills 文件夹；

  2. 自动检测本地已安装的 AI CLI 工具（Claude Code/Codex 等）；

  3. 可选择导入已有 CLI 配置，快速生成默认供应商。

三、界面总览

主界面分为 4 大核心区域，简洁直观、上手无门槛：

  1. 左侧应用切换区：选择要管理的 AI CLI（Claude/Codex/Gemini 等）；

  2. 顶部功能栏：添加供应商、MCP、提示词、Skills、设置按钮；

  3. 中间供应商列表区：显示已添加供应商（名称、状态、延迟），支持拖拽排序；

  4. 底部状态栏：显示本地代理状态、工具运行状态。

四、核心功能：供应商管理（一键切换）

4.1 添加供应商

  1. 选择左侧目标 AI CLI（如 Claude Code）；

  2. 点击右上角添加供应商按钮；

  3. 选择预设供应商（推荐，自动填充接口地址）或自定义配置；

  4. 填写关键信息：

    • 名称：自定义标识（如 “通义千问 - 阿里云”）；

    • API Key：你的供应商密钥（必填）；

    • 基础 URL：接口地址（预设自动填充，自定义需手动填写）；

    • 模型：选择默认调用模型（如 claude-3.5-sonnet）；

  5. 点击保存，自动写入对应 CLI 配置文件。

4.2 切换供应商

方式一：主界面切换

  1. 在供应商列表中选中目标供应商；

  2. 点击启用按钮，状态变为 “当前激活”；

  3. 生效：Claude Code 无需重启，其他 CLI 重启终端即可。

方式二：托盘快速切换

  1. 最小化窗口后，CC Switch 常驻系统托盘；

  2. 右键托盘图标，直接选择供应商，一键切换。

4.3 供应商管理技巧

  • 测速排序：列表显示供应商延迟，优先选择低延迟节点；

  • 批量导入导出：支持备份所有供应商配置，换机直接恢复；

  • 共享配置：编辑供应商时提取公共配置（如插件参数），一键同步到新供应商；

  • 禁用 / 隐藏：暂时不用的供应商可禁用，减少列表干扰。

五、扩展功能：MCP/Skills/ 提示词管理

5.1 MCP 服务器管理（统一配置）

MCP（Model Context Protocol）用于扩展 AI 工具能力（如调用本地文件、数据库）。

  1. 点击顶部MCP按钮；

  2. 点击添加服务器，选择模板或自定义配置；

  3. 配置服务器命令、参数，开启 / 关闭跨应用同步；

  4. 支持 Deep Link 一键导入他人共享的 MCP 配置。

5.2 Skills 插件管理（一键安装）

Skills 是 AI 工具的功能插件（如代码补全、文档生成）。

  1. 点击顶部Skills按钮；

  2. 浏览官方 / 社区仓库，搜索目标插件；

  3. 点击安装，自动同步到所有关联 AI CLI；

  4. 支持自定义插件仓库、批量卸载、版本更新检测。

5.3 提示词管理（跨应用同步）

统一管理 [CLAUDE.md](CLAUDE.md)、[AGENTS.md](AGENTS.md)、[GEMINI.md](GEMINI.md) 等全局提示词。

  1. 点击顶部提示词按钮；

  2. 新建 / 编辑提示词（Markdown 编辑器，支持预览）；

  3. 启用提示，自动同步到对应 CLI 的配置文件；

  4. 支持跨应用同步，一次修改，所有工具生效。

六、高级功能

6.1 本地代理与故障转移

  • 本地代理：内置代理服务，支持格式转换、请求转发；

  • 自动故障转移：主供应商异常时，自动切换到备用节点（支持熔断、健康监控）；

  • 应用级代理：可单独代理某个 CLI，灵活控制路由。

6.2 使用统计与成本追踪

  • 用量仪表盘：实时查看请求数、Token 消耗、成本统计；

  • 趋势图表：按日 / 周 / 月筛选，分析使用规律；

  • 自定义定价：手动设置模型单价，精准核算成本。

6.3 云同步与备份

  • 云同步：支持 Dropbox/OneDrive/iCloud/WebDAV，跨设备同步配置；

  • 自动备份：每日自动备份数据库，保留最近 10 个版本；

  • 一键恢复：配置丢失时，直接从备份恢复。

6.4 Web 端部署（无头服务器模式）

除桌面端外，CC Switch 支持 Web 服务器模式，适配云服务器 / 无桌面环境：

  1. 编译 Web 前端与服务器端；

  2. 启动服务器，默认监听 3000 端口；

  3. 浏览器访问 http://服务器IP:3000，输入账号密码（默认 admin，密码自动生成）；

  4. Web 端功能与桌面端完全一致，支持远程管理。

七、常见问题

7.1 切换供应商后不生效

  • Claude Code：确认已启用热切换，无需重启；

  • 其他 CLI：重启终端或工具，重新加载配置；

  • 排查：点击供应商 “测速”，检查 API Key / 地址是否正确。

7.2 提示 “无法删除当前激活供应商”

CC Switch 设计为至少保留 1 个激活配置，避免 CLI 无法使用。需先切换到其他供应商，再删除目标供应商。

7.3 如何切回官方登录

  1. 从预设中添加 “官方登录” 供应商；

  2. 切换到官方供应商，执行 CLI 登出 / 登录流程；

  3. 可自由在官方与第三方供应商间切换。

7.4 配置文件存储位置

  • 数据库：\~/\.cc\-switch/cc\-switch\.db（核心配置）；

  • 备份：\~/\.cc\-switch/backups/；

  • Skills：\~/\.cc\-switch/skills/；

  • 日志：\~/\.cc\-switch/logs/。
